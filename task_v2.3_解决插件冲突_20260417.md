## 新增功能：解决插件快捷键冲突

### 问题描述
用户反馈：修改快捷键后，天正插件在启动时加载自己的快捷键，覆盖了用户修改的快捷键，导致修改无效。

### 解决方案
新增"重新加载快捷键"功能，提供多种解决方法：

#### 方法一：快速重新加载（手动）
在 AutoCAD 中执行 `REINIT` 命令，勾选 PGP 文件重新加载。

#### 方法二：创建自动加载脚本（推荐）
工具会生成 `acad_reload_pgp.lsp` 脚本，添加到 AutoCAD 启动组后：
- 每次启动 AutoCAD 自动执行
- 延迟 1 秒后重新加载 PGP 文件
- 确保在所有插件加载完成后覆盖回来

#### 方法三：禁用天正快捷键
找到天正的 PGP 文件重命名或在天正设置中禁用快捷键。

### 使用方法
```
主菜单:
  ...
  6. 重新加载快捷键 (解决插件冲突)
  0. 退出

选择 6 后:
  1. 快速重新加载（生成 SCR 脚本）
  2. 创建自动加载脚本（启动时自动重新加载）
  3. 查看详细教程
```

### 自动加载脚本原理
```lisp
(defun S::STARTUP ()
  (command "_.DELAY" 1000)      ; 延迟1秒
  (command "_.REINIT" 16)       ; 重新加载PGP文件
  (princ)
)
```
